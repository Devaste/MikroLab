## About

MikroLab is a full-featured network simulator for MikroTik devices that reproduces RouterOS v7 LTS behavior **without virtualization**. It uses a pure simulation engine written in Go, paired with a modern web frontend (React + TypeScript) for visual topology design and device management.

Think of it as **Cisco Packet Tracer, but for MikroTik** вҖ” design complex network topologies, configure virtual routers using the **real CLI**, watch packets traverse through firewall rules and NAT, and automate workflows with production `.rsc` scripts, all from a single binary on your machine.

### What you'll be able to do:

- **Design networks visually** вҖ” drag devices, draw cables, build topologies in a React Flow canvas.
- **Configure via real RouterOS CLI** вҖ” identical syntax, tab completion, script execution.
- **Use WebFig-like GUI** вҖ” tabular editors for all configuration entities.
- **Simulate packet flow** вҖ” ARP, IP routing, firewall, NAT, connection tracking.
- **Debug in real time** вҖ” built-in sniffer, torch, ping with live event streaming.
- **Automate** вҖ” write and test RouterOS scripts safely before deploying to production hardware.
- **Save & share** вҖ” export/import projects in YAML format.

## Architecture Overview

### Module Directory Layout

The backend module hierarchy mirrors the RouterOS path structure. Each configuration module lives under `internal/modules/` and is organized by its RouterOS path:

```
internal/modules/
в”ңв”Җв”Җ ip/
в”Ӯ   в”ңв”Җв”Җ address/            # implements /ip address
в”Ӯ   в”Ӯ   в””в”Җв”Җ schema.json
в”Ӯ   в”ңв”Җв”Җ arp/                # implements /ip arp
в”Ӯ   в”Ӯ   в””в”Җв”Җ schema.json
в”Ӯ   в””в”Җв”Җ (future: cloud/, route/, firewall/, ...)
в”ңв”Җв”Җ interfaces/             # implements /interface (top-level; will be moved under / later)
в””в”Җв”Җ system/                 # implements /system (planned)
```

> **Note:** The filesystem path is purely for code organization вҖ” the module's `Path()` method returns the runtime RouterOS path (e.g., `/ip/address`), which is used by `GetNode()` for tree traversal and is **independent** of where the `.go` files are stored.

## Tech Stack

| Layer | Technology |
|---|---|
| **Backend** | Go 1.22+ |
| **Frontend** | React 18 + TypeScript 5 (Vite 6) |
| **UI Canvas** | React Flow 11 |
| **Terminal** | xterm.js 5 |
| **API** | WebSocket (JSON-RPC with versioning) + HTTP REST |
| **Dependency Management** | Go Modules (backend `go.mod`), npm (frontend `frontend/package.json`) |
| **Testing** | Go testing + testify + `.rsc` scenario tests |
| **License** | [AGPL v3](./LICENSE) |

See [`frontend/package.json`](frontend/package.json) for frontend dependencies (managed with `npm install` / `npm update`) and [`go.mod`](go.mod) for Go dependencies (managed with `go get` / `go mod tidy`). Full third-party license details in [`LICENSE-3RD-PARTY.MD`](LICENSE-3RD-PARTY.MD).

> Available when the first alpha is released.

